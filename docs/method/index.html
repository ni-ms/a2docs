Methodology

Designing the 3D model of a scene from the story board started with choosing a scene which was fairly complex for implementation and also feasible to build.

We decided to go with the Block-C scene because of it's eye-catching view and complexity involving multiple cuboidal and elliptical shapes

Using the reference image, calculatively approximating the positions for rendered shapes and adding the blender made models to OpenGL object, we got this alluring 3D model :

Deciding the 3d frameâ€‹

Among the various frames available from our story board, the most iconic and the important scene was the C block scene where most of our story took place. It had an eye-catching view along with sufficient complexity involving cuboidal and elliptical shapes.

With the scene to be implemented decided, we started dividing our model into subparts. The environment of this frame can be broken down into various abstract shapes , the main building can be broken down into 5 cuboids of different dimensions but being symmetric overall and the dome of the building can be made with ellipse. The stage was coded with another cuboid. The grass and path are made of thin cuboids as well.

Implementingâ€‹

We first started with coding the wireframe of a cube by using 6 faces. The faces have been drawn with an inbuilt function of OpenGL which makes a square when 4 points are given as input. We used the same logic to built a cuboid then.

After we got the wireframe which we wanted we filled colours in each of the sides which resulted in the following figure:

To distinguish each of the faces individually we had to enable shadows on the 3D shape and add a light source which casts the shadow.

Dimensionsâ€‹

From here we went on to build the basic 3D model of the building.The dimensions of the buildings have been chosen caliculatively to maintain the ratios so that the model looks more true to the original self.

The final shapes and dimensions that have been implemented are :

6 cuboids of dimensions (l, b, h):- 2-(3, 4, 7), 2-(3, 4, 5), (9, 4, 7.5), (9, 5, 1) 1 ellipsoid

After finishing the basic structure we added more cuboids for the path and the stage. We also added an ellipse at the top for the dome.

Using the color function we changed the colours of all the objects in the 3D scene.

Camera Movementâ€‹

The camera movement was implemented by using global variables to store the rotation and translation. Then the gltranslatef and glrotatef functions were used to allow for the movement of the camera. Furthermore, as seen in the deep dive to the code, we defined functions such that on the press of a key or the movement and click of the mouse, the camera moves in that particular direction or rotates in the specified manner

 glTranslatef(0, 0, -cam_dist);
    glRotatef(cam_phi, 1, 0, 0);
    glRotatef(cam_theta, 0, 1, 0);
    glRotatef(cam_yaw, 0, 0, 1);
    glTranslatef(cam_pan[0], cam_pan[1], cam_pan[2]);
